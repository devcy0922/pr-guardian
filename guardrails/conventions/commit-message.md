# 커밋 메시지 및 PR 제목 컨벤션

## 형식
```
<type>: [<issue-id>] <subject>
```

## 허용 타입
- `feat`: 새 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `docs`: 문서 수정
- `test`: 테스트 추가/수정
- `chore`: 빌드/설정 변경
- `hotfix`: 긴급 수정

## 규칙
- **Jira 이슈 ID 필수**: subject에 Jira 이슈 ID(예: `[GA-1234]`)를 대괄호와 함께 반드시 기재합니다.
- PR 제목도 동일한 컨벤션(예: `feat: [GA-1234] 기능 추가`)을 따르거나, 브랜치명에 Jira 이슈 ID가 포함되어 있어야 자동 검증을 통과할 수 있습니다.
- subject는 50자 이내
- 마침표 금지
- 명령형 현재시제 (예: "추가", "수정", "제거")
- 본문은 선택사항, 72자 줄바꿈
